    Virtual inheritance in C++ is used to solve the **Diamond Problem** that occurs 
    during multiple inheritance. It ensures that a class derived from multiple 
    parents inherits only ONE shared instance of a common base class.

                       A
                      / \
                     B   C
                      \ /
                       D

    Without virtual inheritance:
        - Class D gets TWO copies of A (one through B, one through C)
        - This causes ambiguity:
              D.obj.A::value   // which A?
        - Leads to duplicated data and conflicts known as the "Diamond Problem".

    With virtual inheritance:
        - B and C virtually inherit from A:
              class B : virtual public A {};
              class C : virtual public A {};
        - D now inherits ONLY ONE common instance of A.
        - Ambiguities disappear and storage is reduced.

    ================================================================================
    ðŸ”¸ Why do we need Virtual Inheritance? (Interview Points)
    ------------------------------------------------------------------------------
    âœ” To avoid duplicate base class copies in multiple inheritance  
    âœ” To prevent ambiguity when accessing members of the top base class  
    âœ” To ensure there is only ONE shared base class subobject  
    âœ” To solve the "Diamond Problem"  
    âœ” To make data members consistent and not duplicated  

    ================================================================================
    ðŸ”¸ How Virtual Inheritance Works Internally

    â€¢ The compiler inserts a virtual base pointer (vbptr) in classes B and C  
    â€¢ Both B and C refer to the SAME A subobject
