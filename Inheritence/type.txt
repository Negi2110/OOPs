/*
    ====================================================================================
    üîπ PUBLIC vs PRIVATE vs PROTECTED INHERITANCE ‚Äî INTERVIEW EXPLANATION (WITH EXAMPLES)
    ====================================================================================

    In C++, the type of inheritance controls HOW the members of the base class are 
    treated in the derived class. This is one of the most commonly asked interview 
    questions because it tests knowledge of access specifiers and OOP principles.

    ------------------------------------------------------------------------------------
    üî∏ 1Ô∏è‚É£ PUBLIC INHERITANCE  (Most Common)
    ------------------------------------------------------------------------------------
    Meaning: "Derived class IS-A Base class"
    - Public members of Base   ‚Üí remain public in Derived
    - Protected members of Base ‚Üí remain protected in Derived
    - Private members of Base   ‚Üí NOT accessible in Derived

    This is the inheritance used in real-world OOP designs because it models 
    a true "IS-A" relationship.

    Example:
        class Base {
        public:
            int a = 10;
        protected:
            int b = 20;
        private:
            int c = 30;
        };

        class Derived : public Base {
        public:
            void fun() {
                cout << a;   // ‚úî OK (public ‚Üí public)
                cout << b;   // ‚úî OK (protected ‚Üí protected)
                // cout << c; // ‚ùå ERROR (private is never inherited)
            }
        };

    Interview takeaway:
    ‚úî Public inheritance preserves original access levels  
    ‚úî Models IS-A relationship  
    ‚úî Most widely used form of inheritance  


    ------------------------------------------------------------------------------------
    üî∏ 2Ô∏è‚É£ PRIVATE INHERITANCE
    ------------------------------------------------------------------------------------
    Meaning: "Derived class HAS-A Base class implementation"
    - Public members of Base   ‚Üí become private in Derived
    - Protected members of Base ‚Üí become private in Derived
    - Private members of Base   ‚Üí NOT accessible in Derived

    Private inheritance does NOT express an "IS-A" relationship.
    Instead, it expresses "Implemented-in-terms-of".

    Example:
        class Base {
        public:
            int a = 10;
        protected:
            int b = 20;
        };

        class Derived : private Base {
        public:
            void fun() {
                cout << a;   // ‚úî OK (public ‚Üí private ‚Üí accessible inside class)
                cout << b;   // ‚úî OK (protected ‚Üí private ‚Üí accessible inside class)
            }
        };

        int main() {
            Derived d;
            // cout << d.a;  // ‚ùå ERROR: 'a' is now private in Derived
        }

    Interview takeaway:
    ‚úî Converts base class public/protected members into private  
    ‚úî Used when you want to reuse implementation, not expose interface  
    ‚úî Not an IS-A relationship  


    ------------------------------------------------------------------------------------
    üî∏ 3Ô∏è‚É£ PROTECTED INHERITANCE
    ------------------------------------------------------------------------------------
    (Less common, but interviewers LOVE to ask)

    - Public members of Base   ‚Üí become protected in Derived
    - Protected members of Base ‚Üí remain protected
    - Private members of Base   ‚Üí NOT accessible

    The derived class can use base members internally, but outside 
    users cannot access them.

    Example:
        class Base {
        public:
            int a = 10;
        protected:
            int b = 20;
        };

        class Derived : protected Base {
        public:
            void fun() {
                cout << a;   // ‚úî OK (public ‚Üí protected)
                cout << b;   // ‚úî OK (protected ‚Üí protected)
            }
        };

        int main() {
            Derived d;
            // cout << d.a; // ‚ùå ERROR: 'a' is protected in Derived
        }

    Interview takeaway:
    ‚úî Good when you want to inherit but hide all base class interface  
    ‚úî Still not an IS-A relationship externally  


    ====================================================================================
    üî• FINAL INTERVIEW SUMMARY (Must Memorize)
    ====================================================================================

    PUBLIC INHERITANCE:
        Base public    ‚Üí Derived public  
        Base protected ‚Üí Derived protected  
        Base private   ‚Üí Not accessible  
        ‚úî IS-A relationship  
        ‚úî Most common  

    PROTECTED INHERITANCE:
        Base public    ‚Üí Derived protected  
        Base protected ‚Üí Derived protected  
        Base private   ‚Üí Not accessible  
        ‚úî Used when you want inheritance but hide public interface  

    PRIVATE INHERITANCE:
        Base public    ‚Üí Derived private  
        Base protected ‚Üí Derived private  
        Base private   ‚Üí Not accessible  
        ‚úî Used for implementation reuse  
        ‚úî HAS-A relationship, NOT IS-A  

    ====================================================================================
