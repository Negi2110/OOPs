# âœ… **Assertions â€” Turning Them Off Using `NDEBUG`**

Assertions (`assert(condition)`) are meant **only for debugging**, not for production.
C++ provides a mechanism to **disable all assertions** in the program using:

```cpp
#define NDEBUG
```

This macro must appear **before** including `<cassert>` (or `<assert.h>`).

### âœ” When `NDEBUG` is defined:

* All `assert()` checks are **removed by the compiler**.
* No condition is checked at runtime.
* The program **never stops**, even if assertions fail.

### âœ” When `NDEBUG` is *not* defined:

* `assert()` statements are active.
* Failed assertions print an error message and terminate the program.

This allows developers to have **strict error checking in debug builds**, while **removing runtime overhead** in release builds.

# ğŸ¯ **Typical Uses of Assertions**

Assertions are used for **internal sanity checks**, such as:

* validating parameters inside functions:

  ```cpp

  ```
* verifying array bounds (in debug mode)
* checking invariants in classes
* ensuring preconditions/postconditions in algorithms
