**Encapsulation** is one of the core principles of OOP. It refers to **wrapping data (variables) and functions (methods) into a single unit**â€”the classâ€”and **restricting direct access** to the internal state of an object.

### **1. Member variables define the objectâ€™s state**

Every object maintains its own internal state through **member variables**.

### **2. Why we avoid direct access to variables**

If we allow direct access to member variables:

* It can create **inconsistencies** in the object's state
* It leads to **tight coupling** between the class and the external code using it
* Any change in variable naming or validation rules may break code everywhere

So, giving direct access to the internal state is considered bad design.

### **3. Good practice: Keep data private**

The best practice is:

* **Make data members private**
* Provide **public getter and setter methods** to control access

This ensures:

* Validation before modification
* Controlled read/write access
* Loose coupling
* Better maintainability

### **5. `const` makes a function an accessor**

Appending `const` to a member function guarantees:

* The function **cannot modify any member variables**
* The function becomes a **read-only accessor**
