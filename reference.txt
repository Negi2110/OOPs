### **1. What is a reference?**

* A reference is an **alias for an existing variable**.
* It does **not** create a new variable; it becomes another name for the original one.

---

### **2. How does it behave?**

* Changing the value through the reference **changes the original variable**.
* Both the reference and the original variable refer to **the same memory location**.

---

### **3. Common use cases**

* Mainly used to **pass objects or variables to functions** without creating a copy.
* Very useful when a function needs to **modify the argument** passed to it.

---

### **4. Syntax**

* A reference type uses **&** after the type:

  ```cpp
  int x = 10;
  int &ref = x;
  ```

---

### **5. Initialization Rules**

* A reference **must be initialized at the moment it is declared**.
* References **cannot** be initialized to `NULL` or `0`.
* Except for function parameters and return types, you **must bind it immediately**.

---

### **6. Reference cannot be reseated**

* Once a reference is bound to a variable, it **cannot refer to another variable**.
* However, the **value** of the referenced variable **can change**.
