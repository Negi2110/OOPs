# âœ… **Creating a Thread in C++ Using Win32 API (CreateThread)**

Before C++11 introduced `std::thread`, Windows provided its own function, **`CreateThread()`**, to create new threads at the operating-system level. This function launches a new thread and tells Windows which function should run inside that thread.

---

# ðŸ”§ **Function Prototype (Simplified Explanation)**

```cpp
HANDLE CreateThread(
    LPSECURITY_ATTRIBUTES secAttr,   // security settings (usually NULL)
    SIZE_T stackSize,                // thread stack size (0 = default)
    LPTHREAD_START_ROUTINE threadFunc, // function the thread will run
    LPVOID param,                    // parameter passed to threadFunc
    DWORD flags,                     // thread creation options (0 = run immediately)
    LPDWORD threadID                 // returns the thread ID
);
```
+---------------------------+---------------------------------------------------------------+
| Argument Type             | Description                                                   |
+---------------------------+---------------------------------------------------------------+
| LPSECURITY_ATTRIBUTES     | A pointer to a set of thread security attributes that control |
|                           | the access to the thread. Default attributes are applied if   |
|                           | this value is set to NULL.                                    |
+---------------------------+---------------------------------------------------------------+
| SIZE_T                    | Size in bytes of the threadâ€™s stack. If set to zero the size  |
|                           | will match that of the creating threadâ€™s stack.               |
+---------------------------+---------------------------------------------------------------+
| LPTHREAD_START_ROUTINE    | Pointer to a callback function that runs the new thread.      |
+---------------------------+---------------------------------------------------------------+
| LPVOID                    | Parameters                                                    |
+---------------------------+---------------------------------------------------------------+
| DWORD                     | Flag indicating execution state of thread. If set to zero     |
|                           | the thread executes immediately.                              |
+---------------------------+---------------------------------------------------------------+
| LPDWORD                   | The address of an int variable that holds the thread ID.      |
+---------------------------+---------------------------------------------------------------+


### âœ” Key idea

`threadFunc` is **your function** that runs inside the new thread.

# ðŸ§µ **Example From Slide**

```cpp
DWORD threadId;   // Will store the ID of the new thread
HANDLE hdl;       // Handle to the new thread

hdl = CreateThread(
        NULL,             // Default security
        0,                // Default stack size
        MyThreadFunction, // Function to run in the new thread
        NULL,             // No parameters passed
        0,                // Run thread immediately
        &threadId         // Store thread ID here
    );
```

---

# ðŸ§  **What This Code Does**

1. `CreateThread` is called.
2. Windows creates a **new operating-system thread**.
3. That new thread begins executing the function:

   ```cpp
   DWORD WINAPI MyThreadFunction(LPVOID param)
   ```
4. The `threadId` variable receives the unique ID of the new thread.
5. `hdl` (the handle) can be used to:

   * wait for the thread,
   * close it,
   * or manage it further.

---

# ðŸ”¥ **Important Notes (Interview-Ready)**

* `CreateThread()` is a **low-level Windows API**, powerful but complex.
* The thread function must follow the signature:

  ```cpp
  DWORD WINAPI FunctionName(LPVOID param);
  ```
* `HANDLE` must be closed using `CloseHandle(hdl)` when no longer needed.
* This method works **only on Windows**.
* Modern C++ uses `std::thread` (portable, simpler), but Win32 threads are still used in game engines, system programming, and high-performance apps.

# âœ… **The Thread Function â€” Explanation (One Paragraph)**

In Windows multithreading, the function executed by a newly created thread must follow a specific signature: `DWORD WINAPI threadfunc(LPVOID param)`. 
This function acts like the threadâ€™s version of `main()` and serves as the entry point for the new thread. The thread continues running until this function returns, and any value returned becomes the threadâ€™s exit code. 
If you need to pass data to the thread, it must be passed through the `param` argument, which is a generic pointer (`LPVOID`) that you can cast to any required type inside the thread. This mechanism allows threads to receive input while maintaining a consistent, OS-compatible function signature.
