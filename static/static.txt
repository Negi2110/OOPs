### âœ” What is a static data member?

A **static data member**:

* Belongs **to the class**, not to any specific object
* Only **one copy** exists in memory for the entire class
* Shared by **all objects** of that class

---

# ðŸ§  Key Properties of Static Members

### âœ” 1. Shared across all objects

If 10 objects are created, they **all access the same static variable**.

### âœ” 2. Must be defined outside the class

Because static variables are stored separately in memory.

### âœ” 3. Not initialized inside the constructor

Constructor runs per-object.
Static members must be initialized **once only**.

### âœ” 4. Can be accessed without creating an object

Using:

# âœ… **Code Example (Fully Explained)**

```cpp
#include <iostream>
using namespace std;

class Player {
public:
    // Static data member shared by ALL Player objects
    static int instances;

    // Constructor increments the static counter
    Player() {
        instances++;  
        cout << "Player created. Total = " << instances << endl;
    }
};

// Definition (and optional initialization) of the static member
// Required in C++! Storage allocated here.
int Player::instances = 0;

int main() {

    Player p1;   // instances = 1
    Player p2;   // instances = 2
    Player p3;   // instances = 3

    // Access static member without object
    cout << "Final instance count = " << Player::instances << endl;

    return 0;
}
```

# ðŸ” Why static variable is not initialized in constructor?

Because:

* Constructor runs every time an object is created.
* Static variables should be initialized **only once**.
* So initialization must be done **outside the class**.

```cpp
int Player::instances = 0;  // correct initialization
```
